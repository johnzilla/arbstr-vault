**POST /internal/settle** (per BILL-05, BILL-06, BILL-07, BILL-08, D-03, D-04, D-05, D-08):

Zod body schema:
```
reservation_id: z.string()
actual_msats: z.number().int().positive()
tokens_in: z.number().int().nonnegative()
tokens_out: z.number().int().nonnegative()
provider: z.string()
latency_ms: z.number().int().nonnegative()
```

Response schemas:
- 200: `{ settled: true, refunded_msats: z.number(), actual_msats: z.number(), remaining_balance_msats: z.number() }`
- 404: `{ error: { code: z.string(), message: z.string() } }`

Handler logic:
1. Look up RESERVE entry: `ledgerRepo.findById(app.db, reservation_id)`. If not found or entry_type !== 'RESERVE', return 404 with `{ error: { code: 'not_found', message: 'Reservation not found' } }`.
2. Idempotency check: `ledgerRepo.findByRefIdAndType(app.db, reservation_id, 'RELEASE')`. If found, this settle already happened — compute remaining balance via `ledgerRepo.getBalance(app.db, reserve.agent_id)` and return 200 with `{ settled: true, refunded_msats: Math.abs(reserve.amount_msat) - actual_msats, actual_msats: request.body.actual_msats, remaining_balance_msats }`.
3. Atomic transaction: `app.db.transaction(() => { ... })` containing:
   a. RELEASE entry: `ledgerRepo.insert(app.db, { id: generateTransactionId(), agent_id: reserve.agent_id, amount_msat: Math.abs(reserve.amount_msat), entry_type: 'RELEASE', ref_id: reservation_id, mode: 'simulated' })`
   b. PAYMENT entry: `ledgerRepo.insert(app.db, { id: generateTransactionId(), agent_id: reserve.agent_id, amount_msat: -actual_msats, entry_type: 'PAYMENT', ref_id: reservation_id, mode: 'simulated' })`
   c. Audit entry: `auditRepo.insert(app.db, { agent_id: reserve.agent_id, action: 'PAYMENT_SETTLED', amount_msat: actual_msats, ref_id: reservation_id, metadata: { tokens_in, tokens_out, provider, latency_ms } })`
4. After transaction, get remaining balance: `ledgerRepo.getBalance(app.db, reserve.agent_id)`
5. Return 200: `{ settled: true, refunded_msats: Math.abs(reserve.amount_msat) - actual_msats, actual_msats, remaining_balance_msats }`

**POST /internal/release** (per BILL-09, BILL-10, D-06, D-07, D-08):

Zod body schema:
```
reservation_id: z.string()
```

Response schemas:
- 200: `{ released: true }`
- 404: `{ error: { code: z.string(), message: z.string() } }`

Handler logic:
1. Look up RESERVE entry: `ledgerRepo.findById(app.db, reservation_id)`. If not found or entry_type !== 'RESERVE', return 404 with `{ error: { code: 'not_found', message: 'Reservation not found' } }`.
2. Idempotency check: `ledgerRepo.findByRefIdAndType(app.db, reservation_id, 'RELEASE')`. If found, return 200 with `{ released: true }`.
3. Insert RELEASE entry: `ledgerRepo.insert(app.db, { id: generateTransactionId(), agent_id: reserve.agent_id, amount_msat: Math.abs(reserve.amount_msat), entry_type: 'RELEASE', ref_id: reservation_id, mode: 'simulated' })`
4. Return 200: `{ released: true }`

Note: Both settle and release share the same idempotency guard (checking for RELEASE entry with ref_id = reservation_id). This means once a reservation is settled, attempting to release it returns `{ released: true }` (the RELEASE already exists). This is correct behavior per D-08/D-09.

Error shape for all errors: `{ error: { code: string, message: string } }` per D-11.
