  <action>
    1. Create `src/plugins/pino.plugin.ts` — Export `loggerConfig` object for Fastify's logger option:
       ```typescript
       export const loggerConfig = {
         level: config.LOG_LEVEL,
         redact: {
           paths: [
             'req.headers.authorization',    // HTTP request Authorization header
             'req.headers["authorization"]',  // alternate bracket notation
             '*.token',                       // any nested .token field
             '*.token_hash',                  // stored hash fields
             '*.raw_token',                   // in case raw token passes through
           ],
           censor: '[REDACTED]',
         },
         // In test mode, suppress logs to keep test output clean
         ...(config.NODE_ENV === 'test' ? { level: 'silent' } : {}),
       };
       ```

    2. Update `src/app.ts` — Replace inline logger config with `loggerConfig` import:
       - `const app = Fastify({ logger: loggerConfig })` instead of `{ logger: { level: config.LOG_LEVEL } }`
       - Register `pino-http` AFTER logger config is set (Fastify's built-in pino-http uses the logger config automatically, so just ensure the logger option uses the redact config)

    3. Create `tests/integration/security.test.ts`:

       **Token Redaction Tests:**
       - Test: Capture Pino log output during a request with Authorization header. Verify the log output contains '[REDACTED]' instead of the actual vtk_ token.
         - Approach: Create a custom Pino destination (writable stream) that captures log lines. Build app with that stream as the logger destination. Make a request with a vtk_ token. Assert no log line contains the raw token string. Assert at least one log line contains '[REDACTED]'.
       - Test: Token hash values in log entries do not expose raw vtk_ tokens.
       - Test: Error responses do not leak token values in the response body.

       **Balance Isolation Tests (AGNT-02):**
       - Test: Register agent A and agent B. Deposit 100_000 to A, deposit 50_000 to B. Agent A pays 10_000. Verify: Agent A balance = 90_000. Agent B balance = 50_000 (unaffected). This proves sub-accounts are truly isolated.
       - Test: Register agent A and agent B. Agent A makes a payment. Verify agent B's history does NOT contain agent A's audit entries.
       - Test: Register agent A and agent B. Verify agent A cannot read agent B's balance (403). Verify agent B cannot read agent A's history (403).

       **Fail-Closed Tests (SEC-06):**
       - Test: When the database connection is unavailable (close the SQLite connection mid-test), a payment request returns DENY/FAILED — not a 500 error that leaks internals. The service should catch and return a structured error.
  </action>
