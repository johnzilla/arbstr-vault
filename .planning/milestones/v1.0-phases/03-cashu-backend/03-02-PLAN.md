2. Add `selectRail` helper function (NOT exported — internal to service):
```typescript
function selectRail(
  amountMsat: number,
  preferredRail: 'lightning' | 'cashu' | undefined,
  thresholdMsat: number,
  hasLightning: boolean,
  hasCashu: boolean,
): 'lightning' | 'cashu' | 'simulated' {
  // If only one rail available, use it
  if (!hasLightning && !hasCashu) return 'simulated';
  if (!hasCashu) return 'lightning';
  if (!hasLightning) return 'cashu';

  // Agent hint overrides automatic routing (per user decision)
  if (preferredRail) return preferredRail;

  // Amount-threshold routing (per user decision):
  // Under threshold -> Cashu, at or above -> Lightning
  return amountMsat < thresholdMsat ? 'cashu' : 'lightning';
}
```

Then attempt the payment. If it fails with a non-stream error (for Lightning) or any error (for Cashu), attempt fallback to the other rail:

```typescript
let walletResult: PaymentResult;
try {
  walletResult = await selectedWallet.pay({ ...walletReq });
  if (walletResult.status === 'FAILED' && canFallback(initialRail, options)) {
    // Primary rail failed — try fallback
    const fallbackRail = initialRail === 'lightning' ? 'cashu' : 'lightning';
    selectedWallet = getWallet(fallbackRail);
    // For Lightning->Cashu fallback: need to RELEASE the Lightning RESERVE first
    // For Cashu->Lightning fallback: CashuWalletBackend already restored proofs on fail
    walletResult = await selectedWallet.pay({ ...walletReq });
    finalRail = fallbackRail;
    fallbackOccurred = true;
  }
} catch (walletErr) {
  // ... existing LightningStreamError handling ...
  // For non-stream errors on Lightning or Cashu errors: attempt fallback
}
```

IMPORTANT: The fallback logic must handle the RESERVE/RELEASE correctly:
- Lightning primary -> Cashu fallback: Lightning RESERVE was already written. If Lightning returned FAILED (not stream error), the RELEASE was already written by the existing failed path. The Cashu fallback starts fresh with its own proof lock.
- Cashu primary -> Lightning fallback: CashuWalletBackend restores proofs on failure. The Lightning fallback needs a new RESERVE entry written.

The dual-rail RESERVE logic is complex. For the fallback case, write the RESERVE entry for the FALLBACK rail inside the fallback path. The initial rail's RESERVE/RELEASE is handled by the existing code.

`export async function initializeCashuBackend(db: Db): Promise<WalletBackend>`:

Step 1: Create and initialize CashuClient
```typescript
const client = new CashuClient(config.CASHU_MINT_URL!);
await client.initialize(); // loadMint — required before any operation
```

Step 2: Crash recovery — process all PENDING cashu operations
```typescript
const pending = cashuRepo.getPendingOperations(db);
for (const op of pending) {
  try {
    if (op.melt_quote_id) {
      // Has melt quote — check with mint for actual status
      const quoteStatus = await client.checkMeltQuote(op.melt_quote_id);
      if (quoteStatus.state === 'PAID') {
        // Melt succeeded — proofs were spent. Release lock, write audit.
        cashuRepo.releaseProofs(db, [op.secret]);
        console.info(`[Cashu startup] Crash recovery: melt ${op.melt_quote_id} confirmed PAID — released lock for ${op.tx_id}`);
      } else if (quoteStatus.state === 'UNPAID') {
        // Melt did not go through — proofs NOT spent. Release lock, restore proofs.
        cashuRepo.releaseProofs(db, [op.secret]);
        // The proofs were already deleted from pool in CashuWalletBackend.pay() before melt.
        // On crash without completion, we need to check if proofs are still unspent.
        // Use NUT-07 to check:
        // (For simplicity in crash recovery, just release the lock.
        //  The proofs are lost if they were deleted before melt.
        //  A more robust approach: store proofs in cashu_pending too.)
        console.warn(`[Cashu startup] Crash recovery: melt ${op.melt_quote_id} UNPAID — releasing lock for ${op.tx_id}`);
      }
      // PENDING state at mint — leave our lock, will retry next startup
    } else {
      // No melt quote ID — crashed before melt call. Release lock.
      cashuRepo.releaseProofs(db, [op.secret]);
      console.warn(`[Cashu startup] Crash recovery: no melt_quote_id for ${op.tx_id} — releasing lock`);
    }
  } catch (err) {
    console.error(`[Cashu startup] Crash recovery error for ${op.tx_id}:`, err);
    // Leave PENDING — will retry next startup
  }
}
```

Step 3: Keyset rotation — swap any proofs from inactive keysets
```typescript
try {
  const keysets = await client.getKeysets();
  const activeKeysetIds = new Set(keysets.filter(k => k.active && k.unit === 'sat').map(k => k.id));
  const allProofs = cashuRepo.getAllProofs(db);

  // Find proofs with inactive keyset IDs
  const inactiveProofs = allProofs.filter(p => !activeKeysetIds.has(p.keyset_id));

  if (inactiveProofs.length > 0) {
    console.info(`[Cashu startup] Keyset rotation: ${inactiveProofs.length} proofs from inactive keysets — swapping`);
    const cashuTsProofs = inactiveProofs.map(p => ({ id: p.keyset_id, amount: p.amount, secret: p.secret, C: p.C }));
    const newProofs = await client.swapProofs(cashuTsProofs);

    // Remove old proofs, insert new ones
    db.transaction((tx) => {
      const tdb = tx as unknown as Db;
      cashuRepo.deleteProofs(tdb, inactiveProofs.map(p => p.secret));
      cashuRepo.insertProofs(tdb, newProofs.map(p => ({
        keyset_id: p.id, amount: p.amount, secret: p.secret, C: p.C,
      })), 'keyset_rotation');

      auditRepo.insert(tdb, {
        agent_id: 'system',
        action: 'CASHU_KEYSET_SWAP',
        metadata: {
          old_keyset_ids: [...new Set(inactiveProofs.map(p => p.keyset_id))],
          new_keyset_id: newProofs[0]?.id,
          proof_count: inactiveProofs.length,
        },
      });
    }, { behavior: 'immediate' });
  }
} catch (err) {
  console.error('[Cashu startup] Keyset rotation check failed:', err);
  // Non-fatal — old keyset proofs still work as inputs for now
}
```
