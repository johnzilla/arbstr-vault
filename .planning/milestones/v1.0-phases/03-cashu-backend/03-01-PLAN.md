**Create src/lib/cashu/cashu.client.ts** — Thin wrapper around cashu-ts Wallet:

```typescript
import { Wallet } from '@cashu/cashu-ts';
import type { Proof, MeltProofsResponse, MintQuoteResponse, MeltQuoteResponse } from '@cashu/cashu-ts';
```

Class `CashuClient`:
- Constructor takes `mintUrl: string` and creates `new Wallet(mintUrl, { unit: 'sat' })`
- `async initialize(): Promise<void>` — calls `await this.wallet.loadMint()`. MUST be called before any other operation (Pitfall 5 from RESEARCH.md: constructor makes no network calls).
- `async createMintQuote(amountSat: number): Promise<MintQuoteResponse>` — returns `this.wallet.createMintQuoteBolt11(amountSat)`. The quote.request is a BOLT11 invoice to pay via LND to fund the proof pool.
- `async mintProofs(amountSat: number, quote: MintQuoteResponse): Promise<Proof[]>` — returns `this.wallet.mintProofsBolt11(amountSat, quote)`.
- `async createMeltQuote(bolt11Invoice: string): Promise<MeltQuoteResponse>` — returns `this.wallet.createMeltQuoteBolt11(bolt11Invoice)`.
- `async meltProofs(meltQuote: MeltQuoteResponse, proofs: Proof[]): Promise<MeltProofsResponse>` — returns `this.wallet.meltProofsBolt11(meltQuote, proofs)`.
- `async swapProofs(proofs: Proof[]): Promise<Proof[]>` — calculates total = sum of proof amounts, calls `this.wallet.send(total, proofs)`, returns `result.keep`. Used for keyset rotation swaps.
- `async checkProofStates(proofs: Proof[]): Promise<Array<{Y: string; state: string; witness?: string}>>` — returns `this.wallet.checkProofsStates(proofs)`. Used for crash recovery (NUT-07).
- `async checkMeltQuote(quoteId: string): Promise<MeltQuoteResponse>` — returns `this.wallet.checkMeltQuoteBolt11(quoteId)`. Used for crash recovery polling.
- `async getKeysets(): Promise<Array<{id: string; active: boolean; unit: string}>>` — returns mint keysets via `this.wallet.getKeySets()` or equivalent. Used for keyset rotation detection.
- `selectProofsToSend(amount: number, proofs: Proof[]): { send: Proof[]; keep: Proof[] }` — calls cashu-ts coin selection. This handles denomination math (1, 2, 4, 8, 16... sats). NEVER hand-roll coin selection (Pitfall 1 from RESEARCH.md).

Export `CashuClient` class and re-export `Proof` type from cashu-ts for downstream consumers.

IMPORTANT: Verify exact method signatures from cashu-ts v3.5.0 at import time. The API changed across major versions (State of the Art section in RESEARCH.md). Use `mintProofsBolt11` not `mintTokens`, use `meltProofsBolt11` not `meltTokens`.

**Create src/modules/payments/wallet/cashu.wallet.ts** — CashuWalletBackend:

Implements `WalletBackend` interface. Constructor takes `CashuClient` and `db: Db` (for proof repo access).

`pay(req: PaymentRequest): Promise<PaymentResult>`:

The melt flow (paying a Lightning invoice via Cashu proofs):
1. Call `cashuClient.createMeltQuote(req.destination)` to get quote (amount + fee_reserve in sat)
2. Calculate `totalNeededSat = meltQuote.amount + meltQuote.fee_reserve`
3. Call `cashuRepo.selectProofs(db, totalNeededSat)` to get proofs from pool
4. Lock proofs: `cashuRepo.lockProofs(db, proofSecrets, req.transaction_id, meltQuote.quote)` — if false (already locked), return FAILED with reason 'double_spend_prevented'
5. Remove selected proofs from pool: `cashuRepo.deleteProofs(db, proofSecrets)`
6. Call `cashuClient.meltProofs(meltQuote, proofsForCashuTs)` — this is the async mint call
7. On success (`result.quote.state === 'PAID'`):
   - Release PENDING lock
   - If `result.change` has proofs, store them back in pool (change from fee_reserve overpayment)
   - Return SETTLED with `mode: 'cashu'`, `cashu_token_id: proofSecrets[0]` (first proof secret as reference), `fee_msat: (meltQuote.fee_reserve - change_amount) * 1000`
8. On failure:
   - Release PENDING lock
   - Re-insert the original proofs back into pool (they were not spent)
   - Return FAILED with `mode: 'cashu'`

CRITICAL: The proof selection + lock + pool removal MUST happen inside a single synchronous better-sqlite3 transaction (same pattern as Lightning RESERVE). The melt call happens OUTSIDE the transaction (async). This is the Cashu equivalent of RESERVE/RELEASE.

IMPORTANT: Cashu amounts are in SATS, not msats. Convert `req.amount_msat` to sats using `Math.ceil(req.amount_msat / 1000)` for proof selection. The WalletBackend interface uses msat, but cashu-ts Wallet uses sats.

Handle the case where proof pool is empty/insufficient: if pool balance is less than needed, the CashuWalletBackend should auto-mint proofs first by creating a mint quote, but this requires Lightning to pay the mint invoice. For now, if the pool is insufficient, return FAILED with reason 'insufficient_cashu_pool'. The auto-mint from Lightning (pool funding) will be wired in Plan 02 when the routing layer has access to both wallets.
