**Install the lightning npm package:**

**Create src/lib/lnd/lnd.client.ts** with three exported functions:

1. **`createLndConnection(opts: { cert: string; macaroon: string; socket: string })`**
   - Calls `authenticatedLndGrpc({ cert, macaroon, socket })` from `lightning` package
   - Returns the `lnd` connection object (type: `AuthenticatedLnd`)
   - IMPORTANT: The `lightning` package is CJS. Test `import { authenticatedLndGrpc } from 'lightning'` first. If ESM named import fails (SyntaxError), fall back to `import { createRequire } from 'module'; const require = createRequire(import.meta.url); const { authenticatedLndGrpc } = require('lightning');`. See Pitfall 4 in RESEARCH.md.

2. **`verifyMacaroonScope(lnd: AuthenticatedLnd): Promise<void>`** (SEC-05)
   - Attempts `getChainBalance({ lnd })` — this requires `onchain:read` permission
   - If the call SUCCEEDS: the macaroon is overprivileged (admin.macaroon or similar). Throw: `Error('FATAL: Macaroon has onchain:read permission — refusing to start. Use payment-scoped macaroon.')`
   - If the call FAILS with permission error: macaroon is correctly scoped. Proceed silently.
   - Re-throw our own FATAL error; catch and ignore LND permission-denied errors only

3. **`connectWithRetry(opts: { cert: string; macaroon: string; socket: string }): Promise<AuthenticatedLnd>`**
   - Calls `createLndConnection` then `verifyMacaroonScope` then `getWalletInfo({ lnd })` (health check, requires `info:read`)
   - Retries up to 5 times with exponential backoff: delays [1s, 2s, 4s, 8s, 16s]
   - On FATAL macaroon error: do NOT retry — exit immediately with `process.exit(1)` after logging
   - On `getWalletInfo` success: check `info.is_synced_to_chain` — if false, log warning but continue (regtest may not need chain sync)
   - After 5 failures: throw Error('LND connection failed after 5 retries — refusing to start')
   - Log each retry attempt with the attempt number and error message via a `logger` parameter or console

Export the `AuthenticatedLnd` type re-export from `lightning` for downstream consumers.

**Type export:** Also export a type alias `LndClient = AuthenticatedLnd` so downstream code has a clean import path.
