  <files>

  <action>
    Create `src/modules/payments/payments.service.ts` — Export `paymentsService` object:

    **`processPayment(db, agentId: string, request: { amount_msat, asset, purpose, destination_type, destination }): Promise<PaymentResponse>`**

    This is the critical function. Implementation:

    1. Generate a transaction ID upfront: `const txId = generateTransactionId()`

    2. Execute everything inside `db.transaction(async (tx) => { ... }, { behavior: 'immediate' })`:

       a. **Read state (inside transaction for consistency):**
          - `const agent = await agentsRepo.getWithPolicy(tx, agentId)` — if null, throw (should never happen — auth verified agent exists)
          - `const balance = await ledgerRepo.getBalance(tx, agentId)`
          - `const dailySpend = await ledgerRepo.getDailySpend(tx, agentId)`

       b. **Evaluate policy (fail-closed):**
          - `const decision = evaluatePolicy(agent.policy, { balance_msat: balance, daily_spent_msat: dailySpend, request_amount_msat: request.amount_msat })`

       c. **Write PAYMENT_REQUEST audit entry (ALWAYS — even for DENY):**
          - `await auditRepo.insert(tx, { agent_id: agentId, action: 'PAYMENT_REQUEST', policy_decision: decision.outcome, policy_reason: decision.reason, amount_msat: request.amount_msat, metadata: { asset: request.asset, purpose: request.purpose, destination_type: request.destination_type } })`

       d. **If decision is not ALLOW:** return `{ transaction_id: txId, policy_decision: decision.outcome, reason: decision.reason, mode: 'simulated', status: 'FAILED' }`. The audit entry is written but no ledger entry — the deny is recorded but no money moves.

       e. **If ALLOW — execute wallet:**
          - `const walletResult = await simulatedWallet.pay({ amount_msat: request.amount_msat, asset: request.asset, purpose: request.purpose, destination_type: request.destination_type, destination: request.destination, transaction_id: txId })`

       f. **Write ledger debit (negative amount):**
          - `await ledgerRepo.insert(tx, { agent_id: agentId, amount_msat: -request.amount_msat, entry_type: 'PAYMENT', ref_id: txId })`

       g. **Write PAYMENT_SETTLED audit entry:**
          - `await auditRepo.insert(tx, { agent_id: agentId, action: 'PAYMENT_SETTLED', amount_msat: request.amount_msat, ref_id: txId })`

       h. **Return success:**
          - `{ transaction_id: txId, policy_decision: 'ALLOW', mode: walletResult.mode, status: walletResult.status }`

    3. Wrap the ENTIRE function body (including the transaction call) in a try/catch. On any error, log the error and return `{ transaction_id: txId, policy_decision: 'DENY', reason: 'internal_error', mode: 'simulated', status: 'FAILED' }`. This is the service-level fail-closed — if the DB itself errors, we deny.

    **Type for PaymentResponse:**
    ```typescript
    interface PaymentResponse {
      transaction_id: string;
      policy_decision: PolicyOutcome;
      reason?: string;
      mode: string;
      status: 'SETTLED' | 'PENDING' | 'FAILED';
    }
    ```

    CRITICAL INVARIANTS:
    - The `{ behavior: 'immediate' }` option on the transaction prevents TOCTOU races (SEC-02)
    - Audit writes are INSIDE the transaction — if they fail, the payment rolls back (OBSV-02)
    - Policy evaluation happens INSIDE the transaction with consistent balance/spend reads
    - The outer try/catch ensures the service never throws — it returns a deny response
  </action>
