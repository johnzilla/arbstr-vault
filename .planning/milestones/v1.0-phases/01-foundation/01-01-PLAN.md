  <action>
    1. Run `npm init -y` in the project root. Set `"type": "module"` in package.json.

    2. Install core dependencies:
       ```
       npm install fastify fastify-type-provider-zod zod pino pino-http @fastify/bearer-auth dotenv drizzle-orm better-sqlite3 ulidx
       ```

    3. Install dev dependencies:
       ```
       npm install -D typescript vitest tsx drizzle-kit @types/node @types/better-sqlite3 pino-pretty
       ```

    4. Add npm scripts to package.json:
       - `"dev": "tsx watch src/index.ts"` (hot reload dev server)
       - `"start": "node --import tsx src/index.ts"` (production start)
       - `"test": "vitest run"` (test runner)
       - `"test:watch": "vitest"` (test watcher)
       - `"db:generate": "drizzle-kit generate"` (generate migrations)
       - `"db:migrate": "drizzle-kit migrate"` (apply migrations)

    5. Create `tsconfig.json` with: target ES2022, module NodeNext, moduleResolution NodeNext, strict true, outDir dist, rootDir src, esModuleInterop true, skipLibCheck true, resolveJsonModule true.

    6. Create `drizzle.config.ts`:
       - dialect: sqlite
       - schema: ./src/db/schema.ts
       - out: ./src/db/migrations

    7. Create `.env.example` with all required env vars:
       ```
       VAULTWARDEN_ADMIN_TOKEN=change-me-to-a-32-char-minimum-secret
       DATABASE_PATH=./vaultwarden.db
       PORT=3000
       LOG_LEVEL=info
       NODE_ENV=development
       ```

    8. Create `.gitignore` including: node_modules, dist, *.db, *.db-wal, *.db-shm, .env (but NOT .env.example), coverage.

    9. Create `src/config.ts` — Zod v4 schema validating all env vars at import time. Use `import { z } from 'zod/v4'`. Include VAULTWARDEN_ADMIN_TOKEN (string, min 32), DATABASE_PATH (string, default './vaultwarden.db'), PORT (coerce number, default 3000), LOG_LEVEL (enum trace/debug/info/warn/error, default 'info'), NODE_ENV (enum development/test/production, default 'development'). Call `configSchema.parse(process.env)` and export the result as `config`. Load dotenv at the top of this file before parsing.

    10. Create `src/types.ts` — Define branded type aliases: `AgentId = \`ag_\${string}\``, `TransactionId = \`tx_\${string}\``, `PolicyId = \`pl_\${string}\``, `TokenId = \`vtk_\${string}\``. Export generator functions: `generateAgentId()`, `generateTransactionId()`, `generatePolicyId()`, `generateAgentToken()`. Each calls `ulid()` from ulidx and prepends the prefix. Also export `hashToken(raw: string): string` using `crypto.createHash('sha256').update(raw).digest('hex')`.

    11. Create `src/modules/payments/wallet/wallet.interface.ts` — Define and export:
        - `PaymentRequest` interface: `{ amount_msat: number, asset: string, purpose: string, destination_type: string, destination: string, transaction_id: TransactionId }`
        - `PaymentResult` interface: `{ transaction_id: TransactionId, status: 'SETTLED' | 'PENDING' | 'FAILED', mode: string, settled_at?: Date }`
        - `WalletBackend` interface: `{ pay(req: PaymentRequest): Promise<PaymentResult> }`
        This is the contract that SimulatedWallet (Plan 03) and LightningWallet (Phase 2) will implement.
  </action>

<task type="auto">
  <name>Task 2: Drizzle schema, database client, migrations, and Fastify app skeleton</name>
  <files>
    src/db/schema.ts
    src/db/client.ts
    src/app.ts
    src/index.ts
  </files>
  <action>
    1. Create `src/db/schema.ts` with all four tables using Drizzle SQLite column types. Import `ulid` from ulidx for `$defaultFn`.

       **agents table:**
       - id: text PK, $defaultFn `ag_${ulid()}`
       - name: text, notNull, max checked at route level via Zod
       - token_hash: text, notNull (SHA-256 hex of vtk_ token — NEVER plaintext)
       - metadata: text({ mode: 'json' }).$type<Record<string, string>>(), nullable
       - created_at: integer('created_at', { mode: 'timestamp_ms' }).$defaultFn(() => new Date()).notNull()

       **policies table:**
       - id: text PK, $defaultFn `pl_${ulid()}`
       - agent_id: text, notNull, references agents.id, unique (one active policy per agent)
       - max_transaction_msat: integer, notNull, default 0 (deny-all on creation)
       - daily_limit_msat: integer, notNull, default 0 (deny-all on creation)
       - created_at: integer timestamp_ms, $defaultFn new Date(), notNull
       - updated_at: integer timestamp_ms, $defaultFn new Date(), notNull

       **ledgerEntries table:**
       - id: text PK, $defaultFn `tx_${ulid()}`
       - agent_id: text, notNull, references agents.id
       - amount_msat: integer, notNull (positive for deposits, negative for payments)
       - entry_type: text({ enum: ['DEPOSIT', 'PAYMENT', 'REFUND'] }), notNull
       - ref_id: text, nullable (links to wallet transaction ID)
       - created_at: integer timestamp_ms, $defaultFn new Date(), notNull

       **auditLog table:**
       - id: text PK, $defaultFn ulid() (no prefix — internal only)
       - agent_id: text, notNull
       - action: text({ enum: ['AGENT_REGISTERED', 'POLICY_UPDATED', 'PAYMENT_REQUEST', 'PAYMENT_SETTLED', 'PAYMENT_FAILED', 'DEPOSIT'] }), notNull
       - policy_decision: text({ enum: ['ALLOW', 'DENY', 'REQUIRE_HUMAN_APPROVAL'] }), nullable
       - policy_reason: text, nullable
       - amount_msat: integer, nullable
       - ref_id: text, nullable
       - metadata: text({ mode: 'json' }).$type<Record<string, unknown>>(), nullable
       - created_at: integer('created_at', { mode: 'timestamp_ms' }).$defaultFn(() => new Date()).notNull()

    2. Create `src/db/client.ts`:
       - Import Database from better-sqlite3 and drizzle from drizzle-orm/better-sqlite3
       - Import all schema tables as `* as schema`
       - Accept database path from config (import from ../config.ts)
       - Create sqlite instance, set pragmas: `journal_mode = WAL`, `foreign_keys = ON`, `synchronous = NORMAL`
       - Export `db = drizzle({ client: sqlite, schema })` and the `Database` type

    3. Run `npx drizzle-kit generate` to produce migration SQL files in src/db/migrations/.

    4. Create `src/app.ts` — Export a `buildApp()` factory function that:
       - Creates a Fastify instance with logger (pino) configured from config.LOG_LEVEL
       - Sets validatorCompiler and serializerCompiler from fastify-type-provider-zod
       - Registers a GET `/health` route returning `{ status: 'ok', timestamp: new Date().toISOString() }`
       - Returns the app instance (does NOT call listen — that happens in index.ts)

    5. Create `src/index.ts` — Import buildApp and config. Call `buildApp()`, then `app.listen({ port: config.PORT, host: '0.0.0.0' })`. Run migration via `drizzle-orm/better-sqlite3/migrator` `migrate()` before starting the server.

    6. Verify the app boots: create a `.env` file from `.env.example` (with a real 32+ char admin token), run `npx tsx src/index.ts`, confirm `/health` returns 200.
  </action>

Kill the background process after. Also verify migration files exist: `ls src/db/migrations/*.sql` shows at least one file.
  </verify>
