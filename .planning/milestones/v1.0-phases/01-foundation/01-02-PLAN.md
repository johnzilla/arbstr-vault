  <action>
    1. Create `src/modules/tokens/tokens.service.ts`:
       - Export `generateToken()`: calls `generateAgentToken()` from types.ts, returns raw `vtk_` token
       - Export `hashToken(raw: string)`: calls the hashToken from types.ts (SHA-256 hex)
       - Export `verifyTokenConstantTime(provided: string, stored_hash: string)`: hashes `provided` then uses `crypto.timingSafeEqual` to compare with `stored_hash`. Return boolean. Handle different length buffers by returning false (never throw).

    2. Create `src/modules/agents/agents.repo.ts`:
       - Export `agentsRepo` object with methods that accept a Drizzle transaction or db instance as first param:
       - `create(db, values: { id, name, token_hash, metadata? })`: inserts into agents table
       - `findById(db, id: string)`: SELECT from agents WHERE id, returns agent or null
       - `findByTokenHash(db, tokenHash: string)`: SELECT from agents WHERE token_hash, returns agent or null
       - `list(db, opts: { cursor?, limit: number })`: SELECT from agents WHERE id > cursor ORDER BY id ASC LIMIT limit+1. Return items (sliced to limit) and has_more boolean.
       - `getWithPolicy(db, id: string)`: SELECT agent JOIN policies ON agent_id, returns agent with embedded policy or null

    3. Create `src/modules/agents/agents.service.ts`:
       - Export `agentsService` object:
       - `register(db, params: { name, metadata? })`: generates agent ID, generates raw token, hashes it, inserts agent via repo, inserts default deny-all policy (max_transaction_msat: 0, daily_limit_msat: 0), inserts AGENT_REGISTERED audit log entry, returns { agent_id, raw_token, created_at }
       - `getById(db, id)`: calls repo.getWithPolicy, returns agent+policy or throws 404
       - `list(db, opts)`: calls repo.list, returns paginated result
       - `updatePolicy(db, agentId, policy: { max_transaction_msat, daily_limit_msat })`: updates policies table for agent, inserts POLICY_UPDATED audit log entry, returns updated policy

    4. Create `src/middleware/adminAuth.ts`:
       - Export an async Fastify `onRequest` hook function
       - Extracts token from `Authorization: Bearer <token>` header
       - Uses `crypto.timingSafeEqual` to compare against `config.VAULTWARDEN_ADMIN_TOKEN`
       - On failure: reply 401 with `{ error: { code: "unauthorized", message: "Invalid admin token" } }`
       - Handle missing header, malformed header, wrong token — all return 401

    5. Create `src/middleware/agentAuth.ts`:
       - Export an async Fastify `onRequest` hook function
       - Extracts token from `Authorization: Bearer <token>` header
       - Rejects if token doesn't start with `vtk_`
       - Hashes the token, looks up agent via `agentsRepo.findByTokenHash()`
       - On success: decorates `request.agentId` with the agent's ID and `request.agent` with the agent object
       - On failure: reply 401 with `{ error: { code: "unauthorized", message: "Invalid agent token" } }`
       - Add Fastify type augmentation: extend FastifyRequest with `agentId?: string` and `agent?: typeof agents.$inferSelect`
       - CRITICAL: Also verify route param `:id` matches the authenticated agent's ID — agents can only access their own resources. If param `id` exists and doesn't match `request.agentId`, return 403 with `{ error: { code: "forbidden", message: "Cannot access another agent's resources" } }`.
  </action>

  </verify>
