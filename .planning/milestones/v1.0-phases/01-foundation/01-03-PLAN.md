  <action>
    1. Create `src/modules/policy/policy.engine.ts`:

       Export types:
       - `PolicyOutcome = 'ALLOW' | 'DENY' | 'REQUIRE_HUMAN_APPROVAL'`
       - `PolicyDecision = { outcome: PolicyOutcome, reason: string, rule_matched?: string }`
       - `PolicyConfig = { max_transaction_msat: number, daily_limit_msat: number }`
       - `PolicyContext = { balance_msat: number, daily_spent_msat: number, request_amount_msat: number }`

       Export function `evaluatePolicy(policy: PolicyConfig | null, ctx: PolicyContext): PolicyDecision`:
       - Wrap entire body in try/catch
       - If policy is null: return `{ outcome: 'DENY', reason: 'no_policy_configured' }`
       - If policy.max_transaction_msat is 0 AND policy.daily_limit_msat is 0: return `{ outcome: 'DENY', reason: 'deny_all_policy' }` (the default deny-all state)
       - If ctx.request_amount_msat > policy.max_transaction_msat: return DENY with reason 'exceeds_max_transaction', rule_matched 'max_transaction_msat'
       - If ctx.daily_spent_msat + ctx.request_amount_msat > policy.daily_limit_msat: return DENY with reason 'exceeds_daily_limit', rule_matched 'daily_limit_msat'
       - If ctx.balance_msat < ctx.request_amount_msat: return DENY with reason 'insufficient_balance'
       - Otherwise: return `{ outcome: 'ALLOW', reason: 'policy_passed' }`
       - catch block: return `{ outcome: 'DENY', reason: 'policy_engine_error' }` — NEVER throw, NEVER return ALLOW

       This function has ZERO external dependencies. No imports except the types defined in the same file. Pure synchronous function.

    2. Create `tests/modules/policy.test.ts` — Unit tests for evaluatePolicy:
       - Test: null policy returns DENY with reason 'no_policy_configured'
       - Test: deny-all policy (both limits 0) returns DENY with reason 'deny_all_policy'
       - Test: amount exceeding max_transaction_msat returns DENY
       - Test: amount exactly at max_transaction_msat returns ALLOW (boundary)
       - Test: daily spend + amount exceeding daily_limit_msat returns DENY
       - Test: daily spend + amount exactly at daily_limit_msat returns ALLOW (boundary)
       - Test: insufficient balance returns DENY
       - Test: all checks pass returns ALLOW
       - Test: policy with NaN or undefined fields returns DENY (fail-closed) — pass a crafted object with NaN values to trigger the catch
       - Test: verify ALLOW is never returned from catch path — pass Object.create(null) as policy to trigger property access error
       - Test: REQUIRE_HUMAN_APPROVAL is a valid outcome type (for Phase 4, document but don't implement the trigger path yet — just verify the type exists)
  </action>

  </verify>

    1. Create `src/modules/ledger/ledger.repo.ts` — Export `ledgerRepo` object:
       - `insert(db, entry: { id?, agent_id, amount_msat, entry_type, ref_id? })`: inserts into ledgerEntries table. If id not provided, auto-generates via schema $defaultFn.
       - `getBalance(db, agentId: string)`: `SELECT COALESCE(SUM(amount_msat), 0) FROM ledger_entries WHERE agent_id = ?`. Returns a single integer (millisatoshis). Balance is ALWAYS derived from SUM, never a mutable counter.
       - `getDailySpend(db, agentId: string)`: `SELECT COALESCE(SUM(ABS(amount_msat)), 0) FROM ledger_entries WHERE agent_id = ? AND entry_type = 'PAYMENT' AND created_at > ?` where `?` is `Date.now() - 86_400_000` (24 hours in ms). Returns integer. This is the rolling 24h window — no midnight reset.
       - `listByAgent(db, agentId: string, opts: { cursor?, limit, entry_type? })`: cursor pagination on ledger_entries.id for an agent. Optional entry_type filter.

    2. Create `src/modules/ledger/ledger.service.ts` — Export `ledgerService` object:
       - `getBalance(db, agentId)`: calls ledgerRepo.getBalance
       - `getDailySpend(db, agentId)`: calls ledgerRepo.getDailySpend
       - `deposit(db, agentId, amount_msat)`: within a `db.transaction({ behavior: 'immediate' })`, inserts a DEPOSIT ledger entry (positive amount) and a DEPOSIT audit log entry. Returns the new balance.
       - `debit(db, agentId, amount_msat, refId)`: inserts a PAYMENT ledger entry (NEGATIVE amount_msat — store as `-amount_msat`). Called from within an existing transaction (accepts tx, not db). Does NOT start its own transaction — the payment service's transaction wraps this.

    3. Create `src/modules/audit/audit.repo.ts` — Export `auditRepo` object:
       - `insert(db, entry: { agent_id, action, policy_decision?, policy_reason?, amount_msat?, ref_id?, metadata? })`: inserts into auditLog table. This is the ONLY exported function that writes — no update, no delete. The append-only invariant is enforced by the module's API surface.
       - `listByAgent(db, agentId: string, opts: { cursor?, limit, action_type?, start_date?, end_date? })`: cursor pagination on auditLog.id. Optional filters for action type and date range (created_at as integer ms). Used by the history endpoint.
